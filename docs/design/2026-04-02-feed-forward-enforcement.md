## Problem Statement

Carabiner currently enforces **feedback** (learns from gate failures after the fact). It does not enforce **feed-forward** (ensure conditions exist before work starts). The gap: agents can bypass quality patterns, skip static analysis, and proceed without spec/plan attestation.

From the harness engineering articles:
- **Rickard (spec layer)**: Spec = durable intent before implementation. Carabiner should enforce spec exists, not implement spec storage.
- **Zernie (feedback loop)**: "The organism: agent → rules → CI → observability → tasks → agent." Carabiner sits at "rules" but doesn't enforce deterministic checks.
- **Fowler (harness)**: Harness = Guides (feed-forward) + Sensors (feedback). Carabiner has sensors, missing guides.

The user wants carabiner to enforce that linting, type checking, and formatting are **non-negotiable** — no path of least resistance for agents.

---

## Core Design Principle

**Carabiner enforces existence, not authorship.**

- Carabiner does not define your spec format (Linear/Jira/markdown)
- Carabiner does not define your lint rules (eslint/ruff/golangci-lint)
- Carabiner **enforces that these things exist and pass**

This maps to how eslint works: eslint doesn't write your code or define your style — it **enforces that SOME style exists and is applied**.

---

## Architecture: Two New Layers

### Layer 1: Enforce (Feed-Forward)

Deterministic static analysis enforcement. `carabiner enforce --all` runs configured tools and **requires exit 0**. No warnings allowed. No bypass.

### Layer 2: Event Log (Workflow Observability)

Every carabiner invocation is logged to SQLite with timestamp, command, args, exit code, duration, and metadata. This creates a workflow audit trail complementary to code observability logs.

## Enforce Layer

### Config: `enforce.yaml`

```yaml
version: 1
tools:
  eslint:
    enabled: true
    args: ["--max-warnings=0", "--no-error-on-unmatched-pattern"]
    files: ["src/**/*.{ts,tsx}", "tests/**/*.{ts,tsx}"]
  
  typescript:
    enabled: true
    command: "tsc --noEmit --strict"
    files: ["src/**/*.{ts,tsx}"]
  
  prettier:
    enabled: true
    args: ["--check", "--ignore-path .prettierignore"]
    files: ["src/**/*.{ts,tsx,css,md}"]
  
  vitest:
    enabled: true
    command: "vitest run --coverage"
    files: ["src/**/*.test.{ts,tsx}"]

behavior:
  fail_on_warning: true
  stop_on_first_failure: false
  parallel: true
```

### CLI: `carabiner enforce`

```bash
carabiner enforce --all                      # Run all enabled tools
carabiner enforce --files src/auth/         # Run on specific files only
carabiner enforce --tool eslint             # Run single tool
carabiner enforce --list                    # List configured tools
```

**Exit codes:**
- 0 = all tools passed
- 1 = one or more tools failed or had warnings

### Pre-Commit Hook

```bash
# Generated by: carabiner init --pre-commit
#!/bin/bash
# .git/hooks/pre-commit
carabiner enforce --all
```

**Bypass resistance**: Agents can `git commit --no-verify`, but CI runs `carabiner enforce --all` and will catch it.

### CI Integration

```yaml
# .github/workflows/carabiner.yml
name: Carabiner Enforce
on: [push, pull_request]
jobs:
  enforce:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install carabiner
        run: go install github.com/donovan-yohan/carabiner/cmd/carabiner@latest
      - name: Run enforce
        run: carabiner enforce --all
```

## Event Log (Workflow Observability)

### SQLite Schema

```sql
CREATE TABLE events (
  id TEXT PRIMARY KEY,           -- UUID
  timestamp DATETIME,             -- ISO8601
  command TEXT,                  -- "enforce", "attest validate", etc.
  args TEXT,                     -- JSON: what files, flags, etc.
  exit_code INTEGER,             -- 0 = pass, 1 = fail
  duration_ms INTEGER,           -- how long it took
  files_touched TEXT,            -- JSON array for enforce commands
  run_id TEXT,                   -- belayer run ID if applicable
  branch TEXT,                   -- git branch
  commit TEXT,                   -- git commit
  metadata TEXT                  -- JSON: extensible
);

CREATE INDEX idx_timestamp ON events(timestamp DESC);
CREATE INDEX idx_run_id ON events(run_id);
CREATE INDEX idx_command ON events(command);
```

### Storage Location

```bash
# Mode determines storage
carabiner init --mode repo    # ~/.carabiner/<repo-slug>/events.db
carabiner init --mode local   # .carabiner/events.db (gitignored)
```

### CLI: `carabiner events`

```bash
carabiner events list                              # Recent events
carabiner events list --command enforce           # Filter by command
carabiner events list --branch feature/auth       # Filter by branch
carabiner events query --files src/auth/          # Query by files
carabiner events trace --run-id run-abc123        # Trace a belayer run
```

### Events Captured Automatically

Every carabiner CLI invocation logs an event — no explicit recording needed:

- `carabiner enforce --all` → event logged
- `carabiner quality check --files src/` → event logged
- `carabiner attest validate` → event logged
- `carabiner init` → event logged

### Event Schema

```json
{
  "id": "evt-abc123",
  "timestamp": "2026-04-02T10:00:00Z",
  "command": "enforce",
  "args": {
    "files": null,
    "all": true,
    "tool": null
  },
  "exit_code": 1,
  "duration_ms": 4523,
  "files_touched": ["src/auth/token.service.ts"],
  "run_id": "run-abc123",
  "branch": "feature/jwt-refresh",
  "commit": "def456",
  "metadata": {
    "tools_failed": ["eslint"],
    "warnings_count": 3
  }
}
```

    files: ["src/**/*.{ts,tsx,css,md}"]

    enabled: true

**Generated files:**
- `.eslintrc.json` (strict preset)
- `tsconfig.json` (strict mode)
- `.prettierrc` (standard config)
- `vitest.config.ts` (with coverage thresholds)

### Go Template

**enforce.yaml:**
```yaml
version: 1
tools:
  golangci-lint:
    enabled: true
    args: ["run", "--tests", "--max-warnings=0"]
    files: ["**/*.go"]
  
  gofmt:
    enabled: true
    args: ["--l", "-d"]
    files: ["**/*.go"]
  
  staticcheck:
    enabled: true
    args: ["."]
    files: ["**/*.go"]

    enabled: true
    args: ["--", "-race", "-coverprofile=coverage.out"]
    files: ["**/*_test.go"]
```

**Generated files:**
- `.golangci.yml` (strict preset)
- `Makefile` targets for each tool

## Implementation Phases

### Phase 1: Core Enforce (MVP)

1. `enforce.yaml` config parsing
2. `carabiner enforce --all` shell out to configured tools
3. Exit code aggregation
4. `enforce.yaml` template for react-typescript
5. `enforce.yaml` template for go

### Phase 2: Pre-Commit + CI (MVP)

1. Pre-commit hook generation (`--pre-commit`)
2. CI workflow generation (`--ci github`)
3. Test the full flow: commit → hook → CI

### Phase 3: Event Log (MVP)

1. SQLite adapter
2. Auto-log every invocation
3. `carabiner events list/query`
4. Storage mode handling (repo vs local)

### Phase 4: Attestations (Future)

1. Spec attestation schema
2. Plan attestation schema
3. `carabiner attest validate` (existence gate)
4. Event log integration

## Success Criteria

1. `carabiner enforce --all` returns exit 0 when all tools pass, exit 1 when any fails
2. `carabiner init --template react-typescript` produces working enforce.yaml + generated config files
3. Pre-commit hook blocks commits when enforce fails
4. CI workflow blocks PRs when enforce fails
5. Every carabiner invocation creates an event in the SQLite database
6. `carabiner events list` shows recent invocations with command, timestamp, exit code

## What This Is NOT

- Not a linter (uses existing linters)
- Not a spec storage system (integrates with Linear/Jira/markdown)
- Not a replacement for belayer (composes with belayer as harness)
- Not auto-evolving (learnings proposed, not auto-committed)
